BundleCli 🚀
כלי CLI פשוט ויעיל לאריזת קבצי קוד לקובץ אחד.

פרויקט זה נוצר כדי להקל על מורים ומרצים לבדוק תרגילי תכנות. במקום לעבור בין עשרות קבצים ותיקיות, הכלי מאפשר לארוז את כל קבצי המקור של הפרויקט לקובץ טקסט יחיד ומאורגן.

תכונות עיקריות 🛠️
סינון לפי שפות: בחירה של שפות ספציפיות או אריזת כל הקבצים (all).

ניקוי קוד: אפשרות להסרת שורות ריקות לחיסכון במקום.

תיעוד מקור: הוספת הערה עם נתיב הקובץ המקורי לכל מקטע קוד.

מיון מותאם: מיון לפי שם קובץ או לפי סוג הקובץ.

חתימת יוצר: הוספת שם כותב הקוד בראש הקובץ המאוחד.

מחולל פקודות (RSP): כלי אינטראקטיבי ליצירת קבצי תגובה להרצה מהירה.

התקנה והרצה ⚙️
דרישות קדם
.NET 6.0 SDK ומעלה.

חבילת System.CommandLine (גרסת Pre-release).

הגדרה ככלי גלובלי (Path)
כדי להריץ את הכלי מכל מקום במחשב:

בצעי Publish לפרויקט.

העתיקו את נתיב תיקיית ה-Publish.

הוסיפו את הנתיב למשתנה הסביבה Path במערכת ההפעלה.

שימוש בפקודות 💻
1. פקודת bundle
אורזת את הקבצים לקובץ אחד.

תחביר דוגמה:

Bash

bundle --language cs --output myBundle.txt --note --sort name --remove-empty-lines --author "Israel Israeli"
אפשרויות (Options): | אפשרות | קיצור (Alias) | תיאור | | :--- | :--- | :--- | | --language | -l | (חובה) רשימת סיומות (למשל cs, py) או all. | | --output | -o | נתיב ושם קובץ היעד. | | --note | -n | הוספת הערה עם נתיב הקובץ המקורי. | | --sort | -s | מיון לפי name (ברירת מחדל) או type. | | --remove-empty-lines | -r | הסרת שורות ריקות מקבצי המקור. | | --author | -a | רישום שם היוצר בראש הקובץ. |

2. פקודת create-rsp
יוצרת קובץ תגובה (.rsp) בצורה אינטראקטיבית. הפקודה תעבור איתך שלב-שלב ותשאל על כל האפשרויות, ובסוף תייצר קובץ שניתן להריץ בקלות.

איך מריצים את הקובץ שנוצר?

Bash

dotnet @myconfig.rsp
